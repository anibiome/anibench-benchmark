# Security and privacy

Use GitHub's private vulnerability-reporting flow under the repository Security
tab. If that flow is unavailable, contact the maintainers without including
exploit details, participant data, credentials, private bucket paths, or
unpublished source text in a public issue.

Release scans cover:

- credentials and access tokens;
- absolute private paths and controlled bucket locations;
- participant, sample, and bridge identifiers;
- raw image, audio, video, omics, and clinical payloads;
- unlicensed source text and private attachments;
- generated archives and Git history.

Evidence locators can point to controlled data; the public registry must never embed the controlled values themselves. A privacy or license ambiguity blocks public bundling until the release receipt is repaired.

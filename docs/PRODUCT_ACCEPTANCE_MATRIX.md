# AniBench product acceptance matrix

This matrix is the release contract for the product, scientific, and
anti-embarrassment requirements that define AniBench. A requirement is not
accepted because prose mentions it; acceptance requires a user-visible artifact
and an executable gate.

## Position and ranking rule

The product requirement “show that this design is second on one benchmark and
fifth on another” is supported only when every compared study has exact geometry
on the same named metric, evidence lane, authority version, and corpus snapshot.
Until that condition holds, AniBench reports the family vector, eligibility
ledger, and Pareto relation or a typed unknown. It never creates an attractive
ordinal rank by substituting zero for missing comparator geometry.

This is not a reduction of the intended product. It is what makes prospective
fundraising comparisons defensible: a Design Preview may lead a comparison on
its declared geometry, but the comparison denominator and every comparator byte
must be inspectable.

## Release interpretation

Software verification, open-source publication, biological calibration, public
leaderboard promotion, patent filing, and Anipedia integration are different
objects. The open-source product can be complete and useful while empirical
calibration continues and stable public ranks remain gated. No later gate is
allowed to rewrite a passing planned-design result or rescale the current leader
to 100%.

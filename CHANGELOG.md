# Changelog

All notable changes are recorded here. AniBench uses semantic versioning for the benchmark contract, not only the Python package.

## 2.0.0-rc.1 - 2026-07-13

- Replaced the legacy mixed-axis score and leaderboard authority with six
  non-interchangeable capacity families: intensive, extensive, longitudinal,
  causal, personalized/sequential, and transport.
- Added the protocol-capacity compiler, protocol-native Pareto optimizer,
  candidate Level-1 authority chain, typed missingness, same-event linkage
  requirements, and repetition/dependence anti-gaming contracts.
- Preserved lifecycle invariance: a planned design is evaluated on declared
  geometry without a numerical maturity penalty, while realized, accessible,
  and demonstrated claims remain separate evidence lanes.
- Added a source-bound external atlas that preserves unknown geometry instead
  of inventing comparator matrices or converting missing facts to zero.
- Added an executable public-repository exporter that copies an exact allowlist,
  excludes private ANI projections and controlled source bodies, scans every
  public byte, and optionally proves a single clean root commit with no remote.
- Replaced v1 release automation with clean-clone, wheel, source-distribution,
  installed-runtime, Studio JavaScript, leakage, deterministic-candidate, SBOM,
  and attestation checks for the current v2 surface.
- Kept public scalar ranking disabled. Software release does not imply
  independent biological calibration, comparator completeness, or a public
  ranking decision.

## Withdrawn pre-v2 history

All pre-v2 benchmark contracts, scientific claims, numeric anchors, study
placements, completion statements, and leaderboard semantics are withdrawn and
are not AniBench authority. They remain available only in Git history for
provenance and audit; no public v2 result may inherit them without a fresh,
source-bound derivation under the current six-family contract.
